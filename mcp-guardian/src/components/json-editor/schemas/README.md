# JSON Schemas for MCP Guardian

This directory contains JSON Schema files automatically generated from the Rust types in the core library. These schemas provide validation, autocompletion, and documentation for JSON configurations in the Monaco Editor.

## Schema Files

- `guard_profile_schema.json` - Schema for Guard Profile entities
- `mcp_server_schema.json` - Schema for MCP Server entities
- `server_collection_schema.json` - Schema for Server Collection entities

## Usage

These schemas are used by the Monaco Editor integration to provide:

1. **Validation** - Immediate feedback on JSON configuration errors
2. **Autocompletion** - Intelligent suggestions for property names and values
3. **Documentation** - Inline help text from schema descriptions
4. **Type Checking** - Verification of value types and formats

Example Monaco Editor integration:

```typescript
import { editor } from 'monaco-editor';
import mcpServerSchema from './schemas/mcp_server_schema.json';

// Configure Monaco Editor with schema
monaco.languages.json.jsonDefaults.setDiagnosticsOptions({
  validate: true,
  schemas: [
    {
      uri: "http://mcp-guardian/schemas/mcp-server.json",
      fileMatch: ["*"],
      schema: mcpServerSchema,
    },
  ],
});
```

## Updates

Do not edit these files directly. They are generated by the `mcp-guardian-schema` package. To update them:

```bash
# Navigate to the schema package
cd /Users/ramos/mcp-guardian/mcp-guardian-schema

# Generate all schemas
cargo run -- export

# Copy to frontend directory
cp schemas/*.json ../mcp-guardian/src/components/json-editor/schemas/

# Or use the Justfile for convenience
just export-to-frontend
```

This will regenerate all schemas from the latest Rust type definitions and copy them to this directory.
